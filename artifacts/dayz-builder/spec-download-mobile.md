# Spec: Download Landing Page + GitHub Actions CI + iOS/Android (Capacitor)

## Problem Statement

Three related deliverables:

1. **Download landing page** — a standalone `download.html` page served alongside the app with big per-OS download buttons (Windows, macOS, Linux, iOS, Android)
2. **GitHub Actions CI** — automatically builds all 3 desktop platforms (Windows `.exe`, macOS `.dmg`, Linux `.AppImage`) on every tagged release and uploads them to GitHub Releases
3. **iOS + Android via Capacitor** — wrap the existing Vite+React app as a native mobile app using Capacitor; build Android APK in this environment; iOS project configured and ready for Xcode on Mac

---

## Feature 1: Download Landing Page

### Location
`public/download.html` — served as a static file by Vite dev server and included in the Electron build's `dist/public/`. Accessible at `/download.html` on the web app URL.

### Design
Matches the existing dark theme (`#0a0804` background, `#d4a017` gold, `#c0392b` red, monospace font). Full-page layout, no React — pure HTML/CSS so it works standalone without the JS bundle.

### Content sections

**Hero:**
- App name "DankDayz Editor" in large gold/red type
- Tagline: "DayZ base builder — works offline, no internet required"
- Version badge (e.g. v1.0.0)

**Download cards — one per platform:**

| Platform | Icon | File | Status |
|----------|------|------|--------|
| Windows | 🪟 | `DankDayz-Editor-Setup-1.0.0.exe` | Links to GitHub Releases |
| macOS | 🍎 | `DankDayz-Editor-1.0.0.dmg` | Links to GitHub Releases |
| Linux | 🐧 | `DankDayz-Editor-1.0.0.AppImage` | Direct download (file in `/public/`) |
| Android | 🤖 | `DankDayz-Editor-1.0.0.apk` | Direct download (file in `/public/`) |
| iOS | 📱 | App Store / TestFlight | "Coming soon" with instructions |

Each card shows:
- Platform name + icon
- File type label (`.exe Installer`, `.dmg`, `.AppImage`, `.apk`, `TestFlight`)
- Download button (gold for available, grey/disabled for coming soon)
- Brief install instructions (1–2 lines)

**"Back to app" link** at top-left.

**Footer:** "Built with Electron + Capacitor · Fully offline after install"

### Link from main app
Add a small `⬇ Download App` link in the App.tsx header (right side, next to the green pulse dot) that opens `/download.html` in a new tab.

## Feature 2: GitHub Actions CI

### Workflow file: `.github/workflows/release.yml`

**Trigger:** `push` to tags matching `v*.*.*` (e.g. `v1.0.1`)

**Jobs — one per platform:**

```
build-linux   → ubuntu-latest  → AppImage + deb
build-windows → windows-latest → NSIS .exe
build-mac     → macos-latest   → .dmg (x64 + arm64)
```

**Each job steps:**
1. `actions/checkout@v4`
2. `pnpm/action-setup@v4` + `actions/setup-node@v4` (Node 22)
3. `pnpm install --frozen-lockfile`
4. `pnpm build:electron` (Vite build with `BASE_PATH=./`)
5. `pnpm electron:compile` (tsc for main process)
6. `electron-builder --[linux|win|mac]` with `GH_TOKEN` env var
7. `actions/upload-artifact` — upload built files

**Release job** (runs after all 3 build jobs):
- Downloads all artifacts
- Creates/updates GitHub Release using `softprops/action-gh-release@v2`
- Attaches all installer files

**Secrets needed:** `GH_TOKEN` (GitHub personal access token with `repo` scope) — documented in README.

### `electron-builder.yml` update
Add `publish: provider: github` (already present) — electron-builder will auto-upload to the release when `GH_TOKEN` is set.

## Feature 3: iOS + Android via Capacitor

### Approach
Capacitor wraps the Vite build output (`dist/public/`) as a WebView-based native app. Same React codebase, no rewrite. The app runs fully offline — all assets are bundled into the native app.

### New dependencies
```
@capacitor/core
@capacitor/cli
@capacitor/ios
@capacitor/android
```

### `capacitor.config.ts`
```ts
import { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.dankdayz.editor',
  appName: 'DankDayz Editor',
  webDir: 'dist/public',
  server: {
    androidScheme: 'https',
  },
  plugins: {
    SplashScreen: {
      launchShowDuration: 0,
    },
  },
};

export default config;
```

### Build flow
```
pnpm build:electron   # Vite build → dist/public/
npx cap sync          # Copy dist/public/ into ios/ and android/ native projects
npx cap open ios      # Open in Xcode (Mac only)
npx cap open android  # Open in Android Studio
```

### Android
- `android/` project generated by `npx cap add android`
- Build APK: `cd android && ./gradlew assembleRelease` (requires Android SDK)
- Output: `android/app/build/outputs/apk/release/app-release-unsigned.apk`
- Rename to `DankDayz-Editor-1.0.0.apk` and copy to `public/` for direct download
- **Note:** Android SDK not available in this environment — the `android/` project will be generated and committed; user builds APK locally with Android Studio or via GitHub Actions

### iOS
- `ios/` project generated by `npx cap add ios`
- Requires Mac + Xcode 15+ to build
- User opens `ios/App/App.xcworkspace` in Xcode, sets their Apple Developer team, archives and distributes via TestFlight or App Store
- **Note:** iOS build cannot be done in this Linux environment — project is generated and committed, ready for Mac

### GitHub Actions — mobile builds
Add two more jobs to `release.yml`:

```
build-android → ubuntu-latest → set up Android SDK → ./gradlew assembleRelease → upload APK
```
iOS build in CI requires a paid macOS runner with Apple certificates — **deferred**. iOS distribution via TestFlight is documented in README instead.

### `package.json` scripts additions
```json
"cap:sync": "pnpm build:electron && npx cap sync",
"cap:android": "npx cap open android",
"cap:ios": "npx cap open ios"
```

## Acceptance Criteria

1. `/download.html` loads standalone (no JS bundle required) and matches the dark theme
2. Linux download button links directly to the AppImage file
3. Windows and macOS buttons link to the GitHub Releases page (placeholder URL until CI runs)
4. Android APK button present; links to APK once built
5. iOS section shows TestFlight instructions / "coming soon"
6. `⬇ Download App` link in App.tsx header opens `/download.html` in new tab
7. `.github/workflows/release.yml` is valid YAML; pushing a `v*` tag triggers all 3 desktop builds
8. `capacitor.config.ts` is valid; `npx cap add ios` and `npx cap add android` succeed
9. `ios/` and `android/` projects are committed and openable in Xcode / Android Studio
10. TypeScript compiles clean

## Implementation Order

1. Create `public/download.html` — full landing page with all platform cards
2. Add `⬇ Download App` link to App.tsx header
3. Install Capacitor: `@capacitor/core`, `@capacitor/cli`, `@capacitor/ios`, `@capacitor/android`
4. Create `capacitor.config.ts`
5. Run `pnpm build:electron` to ensure `dist/public/` exists
6. Run `npx cap add ios` — generates `ios/` project
7. Run `npx cap add android` — generates `android/` project
8. Run `npx cap sync` — copies web assets into both native projects
9. Create `.github/workflows/release.yml` — desktop CI (Linux + Windows + macOS)
10. Add Android build job to the workflow
11. Update `package.json` with `cap:*` scripts
12. Create `README.md` with build + release instructions
13. TypeScript check
